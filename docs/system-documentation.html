Stored Procedures

סטנדרט שמות: sp_ + PascalCase. כל SP בעל @ResultCode ו-@ResultMessage כ-OUTPUT.

SPתיאורResultCodes
sp_UserLoginכניסה למערכת + audit log. חייב לקבל TenantID!0=הצלחה, 1=שגוי, 2=חסום, 3=לא פעיל, 5=חסום ניסיונות
sp_GetUserSaltשליפת Salt לפי TenantID + Username—
sp_OrganizationCreateיצירת ארגון + הגדרות + AuditLog0=הצלחה, 1=קוד תפוס, 2=מייל תפוס, 3=פורמט שגוי, 99=שגיאת שרת
sp_OrganizationUpdateעדכון ארגון קיים + AuditLog0=הצלחה, 1=לא נמצא, 2=מייל תפוס, 3=לא ניתן לנטרל פלטפורמה, 99=שגיאת שרת
sp_OrganizationGetשליפת ארגון בודד + סטטיסטיקות0=הצלחה, 1=לא נמצא
sp_ResetPasswordאיפוס סיסמה על ידי מנהל—
sp_CreateCustomerיצירת לקוח חדש—
sp_CreateTicketיצירת תקלה חדשה—
🔐 אבטחה: משתמש ה-DB צריך הרשאת EXECUTE בלבד על ה-SPs — ללא גישה ישירה לטבלאות.

Backend API — Node.js

שרת: http://localhost:3000 | כל route מתחת ל-/api/* מחזיר Content-Type: application/json

Routes

MethodRouteAuthתיאור
GET/api/public/tenant/:codeללאפרטי ארגון ציבוריים (שם, לוגו, הגדרות מסך כניסה)
POST/api/auth/loginללאכניסה למערכת → JWT token
POST/api/auth/logoutJWTהתנתקות (stateless)
GET/api/tenantsJWT + TenantID=1רשימת כל הארגונים
GET/api/tenants/:idJWT + TenantID=1ארגון בודד (sp_OrganizationGet)
POST/api/tenantsJWT + TenantID=1יצירת ארגון (sp_OrganizationCreate)
PUT/api/tenants/:idJWT + TenantID=1עדכון ארגון (sp_OrganizationUpdate)
GET/api/stats/platformJWT + TenantID=1סטטיסטיקות דשבורד (ארגונים + משתמשים)
POST/api/upload/logoJWTהעלאת קובץ לוגו → שמירה ב-assets/logos/
POST/api/admin/reset-passwordJWT + RoleID=1איפוס סיסמה

JWT Payload

{
  "userId":     1,
  "roleId":     1,       // 1=Platform Admin, 2=Org Admin
  "customerId": 1,
  "tenantId":   1,       // TenantID=1 = Profits (Platform Admin)
  "fullName":   "שלמה שלום דהאן",
  "roleName":   "מנהל מערכת"
}

הצפנת סיסמאות

// SHA256(password + salt) → base64
hash = crypto.createHash('sha256')
  .update(password + salt, 'utf8')
  .digest('base64');
